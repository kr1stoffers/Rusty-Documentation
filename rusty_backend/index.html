§Rusty Backend - Веб-сервер на Actix Web

Это приложение использует Actix Web для создания веб-сервера, который взаимодействует с базой данных PostgreSQL. Основные функции включают управление пользователями, серверами и каналами.

§Основные возможности

  • Управление пользователями: регистрация, авторизация, CRUD операции
  • Управление серверами: создание серверов, добавление участников
  • Текстовые каналы: создание каналов, отправка сообщений
  • Голосовые каналы: поддержка WebRTC для голосового общения
  • Структурированное логирование: использование tracing для мониторинга

§Структура API

§Пользователи

  • POST /user/register - Регистрация нового пользователя
  • POST /user/login - Авторизация пользователя
  • GET /user - Получение списка всех пользователей
  • GET /user/{id} - Получение пользователя по ID
  • PUT /user/{id} - Обновление пользователя
  • DELETE /user/{id} - Удаление пользователя

§Серверы

  • POST /server - Создание нового сервера
  • GET /server - Получение списка всех серверов
  • POST /server/{server_id}/member - Добавление участника на сервер
  • GET /server/{server_id}/member - Получение участников сервера

§Текстовые каналы

  • POST /server/{server_id}/text_channel - Создание текстового канала
  • GET /server/{server_id}/text_channel - Получение каналов сервера
  • DELETE /server/text_channel/{text_channel_id} - Удаление канала
  • POST /server/text_channel/{text_channel_id}/user/{author_id}/message - Отправка сообщения
  • GET /server/text_channel/{id}/user/message - Получение сообщений канала

§Голосовые каналы

  • POST /server/voice_channel/{voice_channel_id}/user/{user_id}/sdp - Отправка SDP для WebRTC
  • POST /server/voice_channel/{voice_channel_id}/user/ - Добавление участника в голосовой канал

§Запуск

# Установка переменной окружения для логирования
export RUST_LOG=info

# Запуск сервера
cargo run

§Конфигурация

  • Порт: 8080 (по умолчанию)
  • База данных: PostgreSQL
  • Логирование: tracing с поддержкой env-filter

§Зависимости

  • Actix Web - Веб-фреймворк
  • SQLx - Асинхронная работа с БД
  • Tracing - Структурированное логирование
  • Serde - Сериализация/десериализация
